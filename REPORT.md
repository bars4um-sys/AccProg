# Отчёт о проделанной работе — AccProg

> Документ для передачи проекта и будущей доработки.
> Дата: июнь 2026. Версия приложения: 1.1.

---

## 1. Что это за проект

**AccProg** — модульное веб-приложение (набор утилит) для бухгалтеров.
Единая оболочка с четырьмя панелями, в которой каждая «функция» — независимый
модуль. Приложение полностью клиентское (работает в браузере), данные никуда
не передаются.

- **Репозиторий:** https://github.com/bars4um-sys/AccProg
- **Рабочий сайт:** https://bars4um-sys.github.io/AccProg/
- **Стек:** HTML5 + CSS3 + Vanilla JS (ES6), библиотека SheetJS (XLSX). Без бэкенда.

## 2. Исходные данные

В начале были два готовых самостоятельных приложения:

| Приложение | Что делало | Технически |
|---|---|---|
| **SearchDiff** | Сравнивало 2 Excel/CSV таблицы, суммируя значения по компаниям, и показывало расхождения | `index.html` + `app.js` + локальный `xlsx.full.min.js` |
| **SearchFall** | Шаг 1: поиск дублей в авансовом отчёте (Дата+Сумма). Шаг 2: сравнение с банком по «Сумме». Со скачиванием результата в Excel | Один файл `excel-compare.html`, XLSX через CDN |

Оба — клиентские, на чистом JS + SheetJS. Это определило архитектуру AccProg.
Исходники сохранены в папке `_legacy/` как референс.

## 3. Что сделано

### 3.1. Архитектура и план
- Изучены оба приложения, выделены общие принципы.
- Составлен план-структура (`PLAN.md`) и блок-схемы в Mermaid (`FLOWCHART.md`).
- Выбран **Вариант B** реализации: статический HTML+JS без сборки (без Ruby/Jekyll),
  но с модульностью на уровне Jekyll-подхода (реестр функций + контракт модуля).

### 3.2. Оболочка (4 панели)
- `index.html` — каркас: левое меню функций + три рабочие панели
  (ввод данных, операции, результат) на CSS Grid.
- `assets/css/app.css` — раскладка и базовые стили, цвета через CSS-переменные.

### 3.3. Ядро и утилиты
- `assets/js/core.js` — реестр функций (`AccProg.register`), построение меню из
  `functions.json`, роутинг (через `location.hash`), формирование контекста `ctx`
  для модулей, рендер кнопок-операций.
- `assets/js/utils.js` — переиспользуемые утилиты: чтение Excel/CSV
  (`readSheetMatrix`, `readSheetObjects`, `parseCSV`), нормализация
  (`normalizeSum`, `normalizeDate`, `detectHeaderRow`, `cleanData`),
  форматирование, рендер таблиц, экспорт в Excel (`exportXlsx`).

### 3.4. Функции (модули)
- `assets/functions/compare-tables.js` — портирован SearchDiff + **добавлен
  экспорт результата в Excel** (в оригинале его не было).
- `assets/functions/advance-vs-bank.js` — портирован SearchFall, расширен до
  трёхшагового workflow (см. 3.8).
- `functions.json` — реестр функций (что показывать в меню и какой скрипт грузить).

### 3.5. Оформление (темы)
По запросу (ретро-дизайн для консервативной аудитории) созданы 4 темы с
переключателем в шапке (выбор сохраняется в `localStorage`):
- современная (по умолчанию);
- `theme-cozy-ledger.css` — «Уютная бухгалтерия» (тёплая палитра, гроссбух);
- `theme-retro-office.css` — «Ретро-офис» (стиль Windows 95);
- `theme-vintage.css` — «Винтажный пастельный» (мягкая пастель, цветочные акценты).

### 3.6. Развёртывание
- `.gitignore` (исключение `.venv`, `.DS_Store` и пр.), `.nojekyll`
  (чтобы Pages отдавал файлы как есть), `README.md`.
- Все пути в коде относительные — корректно работает из подпапки Pages.
- Репозиторий инициализирован, запушен, GitHub Pages включён и работает.

### 3.7. Очистка
- Удалены артефакты: `SearchDiff/.venv` (~10 МБ), все `.DS_Store`.
- Демо-приложения перенесены в `_legacy/` (через `git mv`, история сохранена).

### 3.8. Доработка advance-vs-bank (v1.1)
Добавлен **Шаг 3** — поиск строк Авансового отчёта, не имеющих соответствия
в Банковской выписке (зеркальная операция к Шагу 2).

Итоговые операции функции:

| Шаг | Кнопка | Что ищет | Файл экспорта |
|---|---|---|---|
| 1 | Дубли в Авансовом отчёте | Строки АО с одинаковой парой Дата+Сумма | `dubles_*.xlsx` |
| 2 | Банк → не найдено в АО | Строки Банка, сумма которых отсутствует в АО | `bank_bez_ao_*.xlsx` |
| 3 | АО → не найдено в Банке | Строки АО, сумма которых отсутствует в Банке | `ao_bez_bank_*.xlsx` |

Логика Шага 3: собирается множество нормализованных сумм из Банковской выписки
(столбец 2), затем по каждой строке АО (столбец 5) проверяется наличие суммы
в этом множестве. Строки без совпадения выводятся в таблицу и доступны для скачивания.

## 5. Как добавить новую функцию (для будущей доработки)

1. Создать `assets/functions/<id>.js`, вызвав в нём:
   ```js
   AccProg.register({
     id: '<id>',
     title: 'Название',
     icon: '🔧',
     description: 'Что делает',
     renderInput: function (ctx) { /* поля ввода в ctx.inputEl */ },
     actions: [
       { id: 'go', label: 'Выполнить', run: function (ctx) {
           /* логика; вывод через ctx.setOutput(html);
              экспорт через ctx.utils.exportXlsx(...) */
       } }
     ]
   });
   ```
2. Добавить запись в `functions.json`:
   ```json
   { "id": "<id>", "title": "Название", "script": "assets/functions/<id>.js" }
   ```
3. Всё. Функция появится в меню. Остальной код менять не нужно.

**Контекст `ctx`, доступный модулю:**
`inputEl`, `actionsEl`, `outputEl` (DOM панелей), `state` (хранилище данных
функции), `utils` (общие утилиты), хелперы `setOutput`, `clearOutput`,
`enableAction`, `message`.

## 6. Известные ограничения и идеи на будущее

- **Фиксированные индексы столбцов** в «Авансовый отчёт vs Банк» (Дата — 2-й,
  Сумма АО — 5-й, Сумма Банка — 2-й). Стоит добавить ручной выбор столбцов
  в панели ввода — при другой структуре таблиц функция работать не будет.
- **Локальный запуск** требует http-сервера (из-за `fetch` для `functions.json`):
  `python3 -m http.server 8000`. Открытие через `file://` не сработает.
- **Нет тестов** — логика проверялась вручную. Можно добавить минимальные
  юнит-тесты для `utils.js` (нормализация сумм/дат, агрегация).
- **Темы** переопределяют стили через CSS-переменные и точечные правила; при
  добавлении новых UI-элементов проверять их вид во всех темах.
- Возможные новые функции: объединение таблиц, поиск по реестру, проверка
  контрольных сумм, конвертация форматов и т.п.
